# 第 7 章：RLHF 完整流水线——从模仿到对齐的工程全景

第 6 章我们深入了 PPO——当前最主流的 on-policy RL 算法，它用裁剪机制稳定训练、用 GAE 精确估计优势函数。PPO 是 RLHF 的"引擎"，但仅有引擎不够——如果要用 PPO 从零训练一个"好用的"对话助手，你还需要奖励模型、偏好数据、SFT 数据、训练稳定性控制……这些工程环节才是真正决定成败的。这一章我们就来搭建一条完整的 LLM 后训练流水线。

一个基础模型变成"好用"的对话助手，中间要经历三个阶段：先用 SFT 教它"怎么回答问题"，再用偏好数据训练一个奖励模型教它"什么是好回答"，最后用 RL 让它真正学会"写出好回答"。这三个阶段不是各自为战的——SFT 的数据质量决定了 RM 能学到什么，RM 的准确性决定了 RL 会往哪里走，而 RL 训练的稳定性决定了最终模型能不能用。任何一环出问题，最终的效果都会大打折扣。

本章把这些环节串联成一条完整的工程流水线。我们从理论基础出发，理解 SFT 和 RM 背后的模仿学习原理；然后深入数据工程和奖励函数设计这两个"论文一笔带过、实际工作最耗时"的环节；最后讨论训练稳定性、奖励黑客防范和 RLAIF 等前沿方向——这些都是工业界每天都在面对的真实问题。
