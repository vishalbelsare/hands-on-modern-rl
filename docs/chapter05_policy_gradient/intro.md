# 第 5 章：策略梯度与 Actor-Critic——从摇骰子到双网络协作

上一章我们走完了 Value-Based 路线——先学 Q 值，再从 Q 值推导策略。DQN 在 CartPole 和 Atari 上工作得很好，但有一个根本性的局限：它只能处理离散动作。CartPole 只有"左推"和"右推"两个选项，所以 DQN 可以对每个动作算一个 Q 值然后取最大。但如果你的机器人有几十个关节，每个关节可以施加任意大小的力矩呢？连续动作空间里，你不可能对每个力矩值都算一遍 Q 值。

更关键的是，大模型对齐场景中的"动作"是从几万个 token 中选一个——虽然看起来是离散的，但策略本身是一个连续的概率分布。DQN 的 $\arg\max$ 在这里水土不服。

这就引出了另一条路线：**跳过 Q 值，直接学策略**。不问"每个动作值多少分"，而是直接学"在什么情况下该做什么"。就像你学骑自行车——你不需要先算出"左转 15 度值 0.87 分"，而是直接练出"感觉这样骑更稳"的直觉。这条路线叫做 **Policy-Based 方法**，而本章要讲的策略梯度（Policy Gradient）就是这条路线的理论核心。

策略梯度方法有一个贯穿始终的"暗线"——**高方差是致命伤，所有改进都围绕降方差展开**：

1. REINFORCE 用实际回报 $G_t$ 来更新策略 → 方差巨大，训练像"醉汉走路"
2. 引入基线（Baseline）减掉"运气"带来的噪声 → 方差降低
3. 用价值函数 $V(s)$ 做基线 → 自然引出 Critic 网络
4. Actor + Critic = Actor-Critic 架构 → 第 6-8 章所有算法的骨架

从 REINFORCE 到 Actor-Critic，就是从"只有一个演员在台上瞎试"进化到"演员 + 评论家互相配合"。这个双网络结构是后续 PPO、DPO、GRPO 的共同骨架——不理解它，就无法理解大模型对齐中的任何一个算法。
