# 第3章：RL 通识——理论地图与全局视野

在前两章中，我们做了两件有趣的事：第 1 章让一根木棍在小车上学会了直立不倒，第 2 章让一个小模型学会了在被拒绝时保持礼貌。两次都做成了——CartPole 拿到了高分，DPO 产出了更友好的回复。

但有一个问题一直被悄悄绕过了：**这些算法到底是怎么"学会"的？**

PPO 为什么能在 20000 步之后突然"开窍"？DPO 的损失函数为什么能把偏好信号注入模型参数？要回答这些问题，我们需要一套数学语言来精确描述"学习"的过程。这就是本章的任务。

## 本章要解决什么问题

RL 要解决的核心问题是：**如何选择动作以最大化累积回报？** 围绕这个问题，本章将沿着一条问题驱动的路径展开：

- 怎么量化策略的好坏？→ 期望回报
- 怎么定义游戏规则？→ MDP
- 怎么评估当前局面？→ $V(s)$
- $V$ 怎么算？→ 贝尔曼方程 → DP/MC/TD
- $V$ 的局限：不告诉你选哪个动作 → 两条路线
  - 路线一：$Q(s,a)$——给动作打分，选最高的
  - 路线二：$J(\theta)$——跳过打分，直接优化策略
