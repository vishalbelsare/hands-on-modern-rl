# MDP 五元组、折扣回报与策略

上一节我们用两台老虎机体验了 RL 的基本交互模式：选择动作，获得奖励，循环往复。现在，让我们把"状态、动作、奖励"这些概念提炼成一个精确的数学框架——马尔可夫决策过程（Markov Decision Process, MDP）。

这个框架由俄罗斯数学家安德烈·马尔可夫在 20 世纪初提出"马尔可夫链"的概念发展而来，后来在 1957 年被理查德·贝尔曼系统化为决策过程的数学框架 [^1]。不管你的智能体在下围棋、控制机器人、还是和人类对话，底层都可以用一个 MDP 来建模。

## MDP 五元组：(S, A, P, R, γ)

一个 MDP 由五个要素定义，通常写作 $(S, A, P, R, \gamma)$。

| 符号 | 名称 | 老虎机 | CartPole |
| --- | --- | --- | --- |
| $S$ | 状态集合 | {初始}（单状态） | $\mathbb{R}^4$（4维连续值） |
| $A$ | 动作集合 | {拉A, 拉B} | {左推, 右推} |
| $P$ | 转移概率 | 始终回到同一状态 | 由物理引擎决定 |
| $R$ | 奖励函数 | 中奖 +1，没中 -1 | 存活 +1 |
| $\gamma$ | 折扣因子 | 不涉及（单步游戏） | 通常 0.99 |

**S（状态集合）**——"环境此刻长什么样"。状态是环境在某一时刻的完整描述。在老虎机中，状态只有一个——不管你拉了多少次，每一轮面对的情况都一样。在 CartPole 中，状态是一个 4 维向量 $[x, \dot{x}, \theta, \dot{\theta}]$，分别代表小车的位置、速度、杆子的角度和角速度。状态必须包含做决策所需的所有信息——这就是"马尔可夫性"的含义：只要你看清了当前的状态，你不需要知道之前是怎么走到这个局面的，就能做出最优决策。

**A（动作集合）**——"智能体能做什么"。老虎机中只有"拉 A"和"拉 B"两个离散动作。CartPole 也是两个离散动作（左推/右推）。而在连续控制任务中（如机器人行走），动作可以是实数向量（如关节力矩）。

**P（转移概率）**——"世界如何运转"。$P(s' | s, a)$ 描述了：在状态 $s$ 下采取动作 $a$ 后，环境变成状态 $s'$ 的概率。在老虎机中，不管你拉哪台，状态都不变。但在 CartPole 中，$P$ 由牛顿力学精确决定。大多数真实 RL 问题中，$P$ 是未知的——智能体需要通过交互来"摸索"。

**R（奖励函数）**——"做得好不好"。$R(s, a)$ 告诉智能体这一步拿了多少分。奖励是 RL 中唯一的学习信号。老虎机的奖励很简单（中奖 +1，没中 -1），但在大模型对齐中，"什么样的回答值得高分"本身就是一个研究课题——这正是 RLHF 和 DPO 要解决的核心问题。

**γ（折扣因子）**——"未来值多少钱"。$\gamma \in [0, 1]$ 控制智能体对"延迟满足"的重视程度。下一节我们会详细讨论它的作用。

为了加深直觉，以下是三种截然不同的 RL 任务的五元组对照：

| 要素 | 老虎机 | CartPole | LLM 对话 |
| --- | --- | --- | --- |
| $S$ | {初始}（单状态） | $\mathbb{R}^4$（连续） | 已生成的 token 序列 |
| $A$ | {拉A, 拉B} | {左, 右} | 词表中的下一个 token |
| $P$ | 出奖率（固定但未知） | 物理引擎 | 语言模型自身的采样 |
| $R$ | ±1 | +1/步 | 人类偏好/奖励模型打分 |
| $\gamma$ | —（单步） | 0.99 | 通常不使用（整句评估） |

尽管表面的差异巨大，但它们都是 MDP——都用同一套数学语言描述。这就是 MDP 框架的强大之处。

<details>
<summary>思考题：LLM 生成满足马尔可夫性吗？</summary>

严格来说，LLM 的自回归生成不满足传统意义上的马尔可夫性——下一个 token 的概率不仅取决于当前 token，还取决于整个上文。但我们可以把"已生成的完整序列"定义为状态 $s$，这样状态转移就只依赖于当前状态（完整上文）和动作（下一个 token），从而满足马尔可夫性。

在实际操作中，RLHF 通常把整个生成过程看作一个 episode，在序列结束后才给予奖励。这更接近蒙特卡洛方法，而非逐步 TD 学习。

## 折扣累积回报 G_t

在老虎机中，每轮只拉一次，我们只关心单步的期望回报。但在 CartPole 这样的多步游戏中，智能体关心的不是某一步的奖励，而是**整个时间线上的总收益**。但未来的奖励不能和眼前的奖励一视同仁——就像今天的 100 块钱比明年的 100 块更有价值一样，未来的奖励需要打一个折扣。

折扣累积回报 $G_t$ 的定义是：

$$G_t = r_t + \gamma \, r_{t+1} + \gamma^2 \, r_{t+2} + \gamma^3 \, r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k \, r_{t+k+1}$$

用一个具体例子来感受：假设 CartPole 每步奖励 +1，$\gamma = 0.9$，智能体活了 4 步后杆子倒了：

$$G_0 = \underbrace{1}_{r_0} + \underbrace{0.9 \times 1}_{r_1} + \underbrace{0.9^2 \times 1}_{r_2} + \underbrace{0.9^3 \times 1}_{r_3} = 1 + 0.9 + 0.81 + 0.729 = 3.439$$

同样是 +1 的奖励，但第 3 步的 1 分只值 $0.729$——因为 $\gamma = 0.9$ 连乘三次后，"未来的 1 分"贬值到不到原来的四分之三。$G_t$ 就是从现在开始到游戏结束，把所有奖励按"越远越不值钱"的规则累加起来的总分。

为什么不直接让 $\gamma = 1$？因为在无限期游戏中，如果 CartPole 永远活下去，总奖励会趋于无穷大——数学上就发散了。折扣因子让累积奖励收敛到一个有限值，同时也符合人类直觉：今天的 100 块比明年的 100 块更有价值。

## 策略 π：智能体的"大脑"

现在我们有了游戏规则（MDP 五元组）和目标（最大化 $G_t$），还差一个关键角色——**策略（Policy）**，即智能体根据状态决定动作的规则。策略是智能体的"大脑"，训练的终极目标就是找到最优策略 $\pi^*$。

策略有两种形式：

**确定性策略**对同一个状态永远输出同一个动作：

$$a = \pi(s)$$

"看到这个局面，就走车"——每次都一样。

**随机性策略**输出动作的概率分布：

$$\pi(a | s) = P(a | s)$$

"看到这个局面，70% 概率走车，30% 概率走马"。随机性策略天然兼顾探索——因为总有小概率去尝试非首选动作。在策略梯度方法中，策略几乎总是随机性的。

回到老虎机的例子：策略 1（随机 50/50）就是 $\pi(\text{拉A}) = 0.5, \pi(\text{拉B}) = 0.5$；策略 2（永远拉 A）就是确定性策略 $\pi(s) = \text{拉A}$。

策略的形式化让我们可以把"找最优策略"变成一个数学优化问题。后续的两条路线（Value-Based 和 Policy-Based）给出了两种截然不同的解法。

## 从游戏规则到局面评估

至此，我们已经有了 RL 的完整形式化框架：

- **MDP 五元组** $(S, A, P, R, \gamma)$ 描述了游戏规则
- **折扣累积回报** $G_t$ 定义了"一局游戏总共拿多少分"
- **策略** $\pi$ 定义了"智能体怎么选动作"

有了规则和目标，接下来要回答一个核心问题：**当前的局面对智能体来说到底值多少分？** 这就是价值函数 $V(s)$ 要解决的问题。
