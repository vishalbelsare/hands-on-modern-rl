# 路线一：Q(s,a)——给每个动作打分

上一节我们认识了状态价值函数 $V(s)$：它告诉你一个局面值多少分。但 $V(s)$ 有一个根本性的局限——它评估的是**状态**，而不是**动作**。知道"当前棋局值 80 分"固然好，但你真正需要回答的问题是："这步棋该走车还是走马？"

要回答这个问题，我们需要一个能区分不同动作的工具——动作价值函数 $Q(s,a)$。

## Q(s,a) 的定义

$Q^\pi(s,a)$ 的定义是：在状态 $s$ 下执行动作 $a$，然后遵循策略 $\pi$ 行动，所能获得的期望累积回报：

$$Q^\pi(s,a) = \mathbb{E}_\pi \left[ G_t \mid s_t = s, a_t = a \right]$$

和 $V(s)$ 相比，$Q(s,a)$ 多了一个条件 $a_t = a$——它固定了第一步的动作。用下棋来类比：

- $V(s)$：这个棋局整体胜率是多少？（不区分走哪步）
- $Q(s, \text{出车})$：如果你走了出车这一步，之后的胜率是多少？
- $Q(s, \text{出马})$：如果你走了出马这一步，之后的胜率是多少？

显然，如果你能知道每个动作的 $Q$ 值，决策就变得直截了当——选 $Q$ 值最高的那个动作。

## V 和 Q 的关系

$V(s)$ 和 $Q(s,a)$ 之间有一个简洁的关系式。$V(s)$ 是所有 $Q$ 值的策略加权平均：

$$V^\pi(s) = \sum_{a} \pi(a | s) \cdot Q^\pi(s, a)$$

策略选择各动作的概率 $\pi(a|s)$ 是权重，$Q(s,a)$ 是对应动作的价值。

用老虎机来验证。假设 A 台出奖率 60%，策略是"70% 拉 A，30% 拉 B"：

- $Q(s, \text{拉A}) = 0.6 \times 1 + 0.4 \times (-1) = 0.2$
- $Q(s, \text{拉B}) = 0.4 \times 1 + 0.6 \times (-1) = -0.2$
- $V(s) = 0.7 \times 0.2 + 0.3 \times (-0.2) = 0.08$

验算无误：每次以 70% 概率拉 A（期望 0.2）、30% 概率拉 B（期望 -0.2），混合下来就是 0.08。

从这个关系式可以看出：$Q(s,a)$ 比 $V(s)$ 携带了更多的信息。$V(s)$ 把所有动作混在一起取平均，而 $Q(s,a)$ 把每个动作的贡献单独拎了出来。正因为如此，$Q$ 天然适合做决策。

## 从 Q 到最优策略：argmax

既然 $Q(s,a)$ 告诉你"在状态 $s$ 做动作 $a$ 值多少分"，那么最优策略就是：

$$\pi^*(s) = \arg\max_a \, Q^*(s, a)$$

选 $Q$ 值最高的那个动作。不需要复杂的推理，比较数字大小就够了。

这就是**路线一（Value-Based）**的核心思路，可以用一个类比来理解：

> 想象你在走迷宫，每到一个岔路口，都能看到一块牌子——往左走预计总共能拿 80 分，往右走预计总共能拿 30 分。你当然选往左。这块"牌子"就是 $Q(s,a)$。它不直接告诉你该走哪条路，只告诉你每条路的预期回报，你自己选最高的。

## 这条路线的代表算法

沿着"学 Q → argmax Q → 得到策略"这条路，主要的算法演进是：

| 算法 | 核心思路 | 特点 |
| --- | --- | --- |
| Q-Learning | 用表格存储每个 (状态, 动作) 的 Q 值 | 经典，但只能处理小规模问题 |
| DQN | 用神经网络代替表格 | 从表格走向像素，深度 RL 的里程碑 |
| Double DQN | 修正 DQN 的过估计偏差 | 更稳定 |
| Dueling DQN | 分离状态价值和动作优势 | 样本效率更高 |

这条路线的优势是**打分准确、样本效率高**（off-policy 可以复用旧数据）。但它有两个短板：一是只能处理**离散动作**（连续空间无法 argmax），二是**不擅长探索**（需要 ε-贪婪等人工补丁来强制探索）。

第 4 章将深入这条路线，从 Q-Learning 的表格方法走到 DQN 的深度学习方法。

但这条路线并不是唯一的答案。如果动作空间是连续的，或者我们不想走"先打分再选"这条路，还有没有别的办法？下一节将介绍完全不同的一种思路——跳过打分，直接优化策略本身。[路线二：J(θ)——直接优化策略](./policy-objective)
