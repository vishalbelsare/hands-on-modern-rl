# 第3章：MDP 与价值函数——从老虎机到贝尔曼方程

在前两章中，我们做了两件有趣的事：第一章让一根木棍在小车上学会了直立不倒，第二章让一个小模型学会了在被骂时保持礼貌。两件事都做成了——CartPole 拿到了高分，DPO 产出了更友好的回复。

但有一个问题一直被我们悄悄绕过了：**这些算法到底是怎么"学会"的？**

PPO 为什么能在 20000 步之后突然"开窍"？DPO 的损失函数为什么能把偏好信号注入模型参数？答案藏在同一个地方——一个叫 MDP（马尔可夫决策过程）的数学框架里。这个框架用五个要素描述了所有 RL 问题的全貌：状态、动作、转移概率、奖励、折扣因子。不管你是在下围棋、控制机器人，还是让大模型学会"说好话"——底层都是 MDP。

MDP 告诉你"游戏规则是什么"。但它不告诉你"怎么赢"。要回答这个问题，你需要知道另一件事：当前的局面对你来说值多少分？这就是价值函数——$V(s)$ 评估一个局面，$Q(s, a)$ 评估一个动作。RL 的核心任务，说白了就是估计这些价值。一旦你知道了每个动作值多少分，最优策略就显而易见了：选分数最高的那个。

那么价值怎么算？你不可能真的去"看到未来所有的可能性"。本章会让你亲手发现一个优雅的解决方案——贝尔曼方程。它的核心洞察是：你不需要知道未来每一步会发生什么，只需要知道"当前奖励 + 下一步的价值"就够了。

本章沿着一条以问题驱动路径展开。我们会从一个赌场游戏——两台老虎机——开始，让你亲身体验 RL 最核心的决策困境。然后一步步搭出价值函数、贝尔曼方程、TD Error，最后走向神经网络和深度 RL。每一步的结尾，你都会自然产生一个问题——下一步刚好回答它。
